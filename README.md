#  SynoBot — Telegram-бот для управления Synology / Xpenology

Легкий и быстрый Telegram-бот для мониторинга и управления вашим NAS (Synology или Xpenology) напрямую из мессенджера. Работает нативно через bash, не требует сложных зависимостей (только `jq`, который скачается сам) и взаимодействует с системой через официальные утилиты DSM.

##  Возможности
* **Мониторинг в реальном времени:** Бот автоматически присылает уведомления о системных событиях DSM, статусе пакетов, входах по SSH/Web и падениях Docker-контейнеров.
* **Управление системой:** Просмотр статуса ресурсов (Temp, RAM, диски), проверка S.M.A.R.T., инфа о сетевых интерфейсах.
* **Безопасность:** Включение и выключение SSH одной кнопкой, экстренное отключение Firewall.
* **Docker и Пакеты:** Просмотр статуса, запуск и перезагрузка контейнеров и официальных пакетов прямо из чата.

##  Быстрая установка

Для начала вам понадобятся:
1. **Токен бота:** Создайте бота через @BotFather в Telegram.
2. **Ваш Telegram ID:** Узнайте свой числовой ID через @getmyid_bot.

### Шаг 1. Вход по SSH и получение прав Root
Подключитесь к вашему NAS по SSH под вашей обычной учетной записью администратора:
```bash
ssh ваш_пользователь@IP_вашего_NAS
```
Затем повысьте права до суперпользователя `root` (потребуется ввести пароль от вашей учетки):
```bash
sudo -i
```

### Шаг 2. Загрузка и запуск скрипта
Выполните команду загрузки скрипта:
```bash
wget -qO /root/synobot.sh https://raw.githubusercontent.com/LocalHost-hub/syno_bot/main/syno_bot.sh && chmod +x /root/synobot.sh
```

**Первый запуск (Обязательно!)**
Запустите скрипт вручную, чтобы пройти первоначальную настройку:
```bash
/root/synobot.sh
```
Консоль попросит вас ввести Токен и Ваш ID. Данные сохранятся в конфигурационный файл `/etc/synobot.conf`, после чего бот запустится и пришлет приветственное сообщение в Telegram.

##  Настройка автозапуска (Планировщик задач DSM)

Чтобы бот работал в фоне и запускался сам после перезагрузки NAS:

1. Откройте веб-интерфейс DSM -> Панель управления -> Планировщик задач (Task Scheduler).
2. Нажмите Создать -> Запущенная задача (Triggered Task) -> Скрипт пользователя (User-defined script).
3. На вкладке "Общие":
   * Задача: SynoBot Autostart
   * Пользователь: root (Обязательно!)
   * Событие: Загрузка (Boot-up).
4. На вкладке "Настройки задачи" в поле скрипта вставьте эту строку:
   ```bash
   nohup /root/synobot.sh > /dev/null 2>&1 &
   ```
5. Нажмите OK.
6. Чтобы запустить бота прямо сейчас, выделите созданную задачу в списке и нажмите кнопку "Запустить" на верхней панели.

##  Остановка бота

Если вам нужно временно остановить бота, зайдите по SSH, получите права root (`sudo -i`) и выполните команду, которая убьет все процессы скрипта:
```bash
pkill -f synobot.sh
```

---

##  Полное удаление

Для полного удаления бота из системы (включая его настройки) выполните под пользователем root следующие команды:
```bash
pkill -f synobot.sh
rm /root/synobot.sh
rm /etc/synobot.conf
```
*Также не забудьте удалить задачу автозапуска в Планировщике задач DSM.*
